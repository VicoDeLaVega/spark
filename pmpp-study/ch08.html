8.0 Introduction — Calculs Stencil

Un calcul stencil met à jour chaque élément d'un tableau en fonction de ses voisins selon un motif fixe. On les trouve partout : résolution d'EDPs (méthode des différences finies), simulation de fluides (Navier-Stokes), imagerie médicale, rendu acoustique. Contrairement à la convolution où le "filtre" est appris, les coefficients du stencil découlent de la discrétisation mathématique.

Exemple : Équation de la Chaleur 2D

∂u/∂t = α (∂²u/∂x² + ∂²u/∂y²). Discrétisée par différences finies centrées d'ordre 2 :

u_new[i][j] = u[i][j] + α·dt/dx² · (u[i-1][j] + u[i+1][j] + u[i][j-1] + u[i][j+1] - 4·u[i][j])

8.1 Stencil 2D — 5 Points (Von Neumann)

Le stencil 2D 5-points (centre + 4 voisins cardinaux) est le plus courant. Chaque élément lit 5 valeurs d'entrée et produit 1 valeur de sortie.

// Stencil 2D 5-points naïf
__global__ void stencil2D_naive(float* u, float* u_new,
                                  int rows, int cols,
                                  float c0, float c1, float c2) {
    int j = blockIdx.x * blockDim.x + threadIdx.x;  // colonne
    int i = blockIdx.y * blockDim.y + threadIdx.y;  // ligne

    if (i < 1 || i >= rows-1 || j < 1 || j >= cols-1) return;

    u_new[i*cols+j] = c0 * u[i*cols+j]
                    + c1 * (u[(i-1)*cols+j] + u[(i+1)*cols+j])
                    + c2 * (u[i*cols+(j-1)] + u[i*cols+(j+1)]);
}

8.2 Stencil 3D 7-Points — Simulation de Volumes

La version 3D ajoute les voisins dans la dimension z. Utilisée en géophysique, CFD volumétrique, IRM.

#define RADIUS 1

__global__ void stencil3D_7pt(float* u, float* u_new,
                               int NX, int NY, int NZ,
                               float c0, float cx, float cy, float cz) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    int z = blockIdx.z * blockDim.z + threadIdx.z;

    if (x<1||x>=NX-1||y<1||y>=NY-1||z<1||z>=NZ-1) return;

    int idx = z*NY*NX + y*NX + x;
    u_new[idx] = c0  * u[idx]
               + cx * (u[idx-1]     + u[idx+1])      // voisins x
               + cy * (u[idx-NX]    + u[idx+NX])     // voisins y
               + cz * (u[idx-NX*NY] + u[idx+NX*NY]); // voisins z
}

// Lancement 3D :
dim3 block(8, 8, 8);   // 512 threads
dim3 grid((NX+7)/8, (NY+7)/8, (NZ+7)/8);
stencil3D_7pt<<<grid, block>>>(d_u, d_u_new, NX, NY, NZ, c0, cx, cy, cz);

8.3 Stencil Tuilé avec Shared Memory — 2D

Même principe que la convolution : charger un tile (+ halo) en shared memory pour réutiliser les données entre threads voisins.

#define TILE_X  32
#define TILE_Y  32
#define RADIUS   2  // ordre 4 en différences finies

__global__ void stencil2D_tiled(float* u, float* u_new,
                                   int rows, int cols,
                                   float* coeff) {  // coeff[RADIUS+1]
    int tx = threadIdx.x, ty = threadIdx.y;

    // Shared memory : tile + halo de RADIUS de chaque côté
    __shared__ float smem[TILE_Y + 2*RADIUS][TILE_X + 2*RADIUS];

    int globalCol = blockIdx.x * TILE_X + tx;
    int globalRow = blockIdx.y * TILE_Y + ty;

    // Chargement avec clamping aux bords
    int smRow = ty + RADIUS, smCol = tx + RADIUS;
    int gRow  = globalRow, gCol = globalCol;

    auto safeLoad = [&](int r, int c) -> float {
        r = max(0, min(rows-1, r));
        c = max(0, min(cols-1, c));
        return u[r*cols+c];
    };

    smem[smRow][smCol] = safeLoad(gRow, gCol);

    // Threads bords chargent le halo
    if (tx < RADIUS) {
        smem[smRow][smCol - RADIUS] = safeLoad(gRow, gCol - RADIUS);
        smem[smRow][smCol + TILE_X] = safeLoad(gRow, gCol + TILE_X);
    }
    if (ty < RADIUS) {
        smem[smRow - RADIUS][smCol] = safeLoad(gRow - RADIUS, gCol);
        smem[smRow + TILE_Y][smCol] = safeLoad(gRow + TILE_Y, gCol);
    }
    __syncthreads();

    if (globalRow >= rows || globalCol >= cols) return;

    // Calcul stencil d'ordre 2r (isotrope, stencil en croix)
    float acc = coeff[0] * smem[smRow][smCol];
    for (int r = 1; r <= RADIUS; r++) {
        acc += coeff[r] * (smem[smRow-r][smCol] + smem[smRow+r][smCol]
                         + smem[smRow][smCol-r] + smem[smRow][smCol+r]);
    }
    u_new[globalRow*cols + globalCol] = acc;
}

8.4 Register Tiling dans la Dimension Z (3D)

Pour un stencil 3D, charger un tile 3D complet en smem utiliserait trop de mémoire. L'astuce : charger une seule couche XY en smem et stocker les voisins Z dans des registres. On déplace (shift) la fenêtre Z au fil des itérations.

__global__ void stencil3D_regTile(float* u, float* u_new,
                                     int NX, int NY, int NZ,
                                     float c0, float cx, float cy, float cz) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    if (x >= NX || y >= NY) return;

    // Shared memory pour la couche XY courante
    __shared__ float smem[BLOCK_Y + 2][BLOCK_X + 2];  // avec halo XY radius=1

    // Registres : voisins Z préchargés
    float reg_prev, reg_curr, reg_next;

    // Préchargement des deux premières couches
    reg_prev = u[0*NX*NY + y*NX + x];                   // z=0
    reg_curr = u[1*NX*NY + y*NX + x];                   // z=1

    for (int z = 1; z < NZ-1; z++) {
        reg_next = u[(z+1)*NX*NY + y*NX + x];           // préchargement z+1

        // Calcul stencil 7-points : X,Y depuis smem, Z depuis registres
        u_new[z*NX*NY + y*NX + x] =
              c0 * smem[sy][sx]
            + cx * (smem[sy][sx-1]  + smem[sy][sx+1])   // voisins X — smem
            + cy * (smem[sy-1][sx]  + smem[sy+1][sx])   // voisins Y — smem
            + cz * (reg_prev        + reg_next);          // voisins Z — registres !

        // Shift de la fenêtre Z
        reg_prev = reg_curr;
        reg_curr = reg_next;
        __syncthreads();
    }
}

8.5 Thread Coarsening pour Stencil

Un thread traite plusieurs couches Z consécutives. Réduit l'overhead de synchronisation et permet de réutiliser le halo XY entre couches successives.

// Thread coarsening : 1 thread gère COARSEN couches Z
#define COARSEN 4

__global__ void stencil_coarsened(float* u, float* u_new,
                                     int NX, int NY, int NZ) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    int z_start = blockIdx.z * COARSEN;  // premier z pour ce thread

    if (x >= NX || y >= NY) return;

    __shared__ float smem[BLOCK_Y+2][BLOCK_X+2];

    // Préchargement sliding window initial
    float prev = u[(z_start-1)*NX*NY + y*NX + x];
    float curr = u[z_start*NX*NY + y*NX + x];

    // Boucle sur les COARSEN couches
    for (int dz = 0; dz < COARSEN && (z_start+dz) < NZ-1; dz++) {
        int z = z_start + dz;
        float next = u[(z+1)*NX*NY + y*NX + x];

        // Chargement smem pour couche z
        smem[threadIdx.y+1][threadIdx.x+1] = curr;
        /* ... chargement halos XY ... */
        __syncthreads();

        u_new[z*NX*NY + y*NX + x] = /* stencil */ 0.0f;  // simplifié

        prev = curr; curr = next;
        __syncthreads();
    }
}
// Bénéfice : COARSEN=4 → 4x moins de __syncthreads() et de blocs lancés
//             Surcoût : chaque thread utilise 3× plus de registres

8.7 Temporal Blocking — Amortir sur le Temps

Pour les simulations itératives (t=0, 1, 2, ..., T), on peut "bloquer" aussi dans la dimension temporelle : effectuer plusieurs pas de temps en smem avant d'écrire en DRAM. Réduit la BW d'un facteur ≈ temporal_tile.

// Concept : temporal tile de profondeur 2
// Itération 1 : charge depuis global, calcule t+1 en smem
// Itération 2 : calcule t+2 depuis smem (sans toucher DRAM !)
// Contrainte : le tile doit "rétrécir" de RADIUS à chaque pas de temps
//   → output_tile_t+1 = output_tile_t - 2×RADIUS (halo croissant)
// Pour 2 pas de temps : tile_t+2 = tile_t0 - 4×RADIUS

// Si tile_t0=32 et RADIUS=2 : tile_t+2 = 32 - 8 = 24 éléments de sortie
// Smem nécessaire : tile_t0 × tile_t0 × sizeof(float) × 2 buffers
// = 32 × 32 × 4 × 2 = 8 KB (faisable !)
🎯 Questions d'Entretien — Ch.8 Stencil
  • « Quelle est la différence entre un stencil et une convolution en termes d'implémentation GPU ? » → Mathématiquement similaires. En pratique : stencil a coefficients fixes dérivés de l'EDP (optimisable) et opère souvent en 3D sur de grands volumes. Convolution a des filtres appris et bénéficie de cuDNN/Winograd. Le register tiling Z est spécifique aux stencils 3D.
  • « Expliquez le register tiling dans la dimension Z. » → Au lieu de charger toutes les couches Z en smem (impossible), on garde les couches Z-1, Z, Z+1 dans des registres et défile une couche à la fois. La couche courante est en smem pour la réutilisation XY. Le shift (prev←curr, curr←next) se fait chaque itération Z.
  • « Quel est l'AI d'un stencil 7-points ? » → 13 FLOP (7 mult + 6 add), 7 floats lus = 28 bytes. AI = 13/28 ≈ 0.46 FLOP/byte. Très memory-bound sur tous les GPU. Le temporal blocking peut multiplier l'AI par le nombre de pas de temps traités par itération.
  • « Qu'est-ce que le temporal blocking et pourquoi est-il difficile ? » → Traiter plusieurs pas de temps en une passe en smem pour éviter les allers-retours DRAM. Difficile car le tile doit rétrécir de 2×RADIUS à chaque pas de temps (les bords ont besoin de voisins hors-tile). Implémentation complexe, bénéfice important pour simulations longues.
  • « Pourquoi __syncthreads() est-il si critique pour les stencils ? » → Sans sync, les threads du même bloc pourraient lire des valeurs partiellement mises à jour par leurs voisins. Le smem serait incohérent. La sync garantit que toute la couche est chargée avant tout calcul.
